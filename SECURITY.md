# Security Policy

## Supported Versions
- Actively supported: `main` branch and the latest released version.

## Reporting a Vulnerability
Please report security issues responsibly and privately. Do not open public GitHub issues for security reports.

Preferred channels:
- Open a private report via GitHub “Security advisories” for this repository or contact the maintainers privately via the repository owners.

What to include:
- A description of the issue and potential impact
- Steps to reproduce and affected configuration(s)
- Any available PoC or logs (sanitize sensitive data)

Triage & disclosure:
- We aim to acknowledge reports within 3–5 business days
- We will investigate, confirm, and work on a fix before public disclosure
- Credit will be given to reporters who request it

Thank you for helping keep the ecosystem safe.
